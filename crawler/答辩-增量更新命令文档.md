# 答辩用：增量更新命令文档

这份文档用于现场演示“增量更新”功能。  
项目里的增量更新分两层：

1. 本地增量：根据已有 CSV 和 `object_id` 做行级比对，输出 `new / updated / unchanged`。
2. 源侧增量：Harvard 支持 `lastupdate` 水位线，减少重复抓取。

对应实现位置：

- `crawler/museum_crawler/incremental.py`
- `crawler/museum_crawler/harvard.py`
- `crawler/museum_crawler/smithsonian.py`
- `crawler/museum_crawler/mfa_boston.py`
- `crawler/museum_crawler/cli.py`
- `crawler/run_incremental_update.sh`

## 1. 演示目标

答辩时建议展示三件事：

1. 第一次运行会产出数据文件。
2. 第二次运行同样命令时，只会处理新增或变化数据。
3. 运行日志会留下可追溯记录。

## 2. 运行前准备

进入爬虫目录：

```bash
cd crawler
```

准备环境变量：

```bash
export SI_DATA_GOV_API_KEY="你的史密森尼API_KEY"
export HARVARD_ART_MUSEUMS_API_KEY="你的Harvard_API_KEY"
```

如果不想写入 MySQL，可以显式关闭：

```bash
export MYSQL_HOST=""
export MYSQL_DATABASE=""
```

## 3. 最推荐的答辩演示方式

### 3.1 第一次运行：先生成基线数据

建议先跑一个较小样本，方便现场演示：

```bash
MUSEUMS=harvard CRAWL_LIMIT=20 ./run_incremental_update.sh
```

如果想三馆一起演示：

```bash
MUSEUMS=all CRAWL_LIMIT=20 ./run_incremental_update.sh
```

第一次运行后，重点看下面几个输出：

- `output/harvard_art_museums.csv`
- `output/crawl_runs.jsonl`
- `output/crawl_changes.jsonl`
- `output/crawl_summary.json`
- `output/.crawl_state/`

### 3.2 第二次运行：重复同样命令，展示增量效果

再次执行同样命令：

```bash
MUSEUMS=harvard CRAWL_LIMIT=20 ./run_incremental_update.sh
```

如果源数据没有明显变化，第二次运行通常会看到：

- `unchanged` 明显增多
- `new` 和 `updated` 变少
- `crawl_runs.jsonl` 新增一条运行记录
- `crawl_changes.jsonl` 新增一条变更记录

这一步是答辩里最关键的“增量”展示。

## 4. Harvard 源侧增量演示

Harvard 支持 `lastupdate` 增量。你可以指定一个水位线：

```bash
MUSEUMS=harvard \
HAM_INCREMENTAL_SINCE="2026-06-01T00:00:00" \
CRAWL_LIMIT=20 \
./run_incremental_update.sh
```

说明：

- 如果 `HAM_INCREMENTAL_SINCE` 为空，程序会自动使用现有 CSV 里的 `source_updated_at` 最大值。
- 如果想关闭源侧增量，只做本地快照比对，可以加：

```bash
MUSEUMS=harvard CRAWL_LIMIT=20 HAM_SOURCE_INCREMENTAL=false ./run_incremental_update.sh
```

## 5. 史密森尼 / MFA 的增量演示

### Smithsonian

```bash
MUSEUMS=smithsonian CRAWL_LIMIT=20 ./run_incremental_update.sh
```

### MFA Boston

```bash
MUSEUMS=mfa CRAWL_LIMIT=20 ./run_incremental_update.sh
```

它们主要展示本地增量更新：

- 先读取已有 CSV 快照
- 再按 `object_id` 判断 `new / updated / unchanged`
- 结果追加到运行台账

## 6. 现场查看结果的命令

查看最近一次运行概况：

```bash
tail -n 5 output/crawl_runs.jsonl
```

查看变更日志：

```bash
tail -n 5 output/crawl_changes.jsonl
```

查看摘要：

```bash
cat output/crawl_summary.json
```

查看某个馆的 CSV 是否被更新：

```bash
ls -lh output/*.csv
```

## 7. 答辩时可以怎么讲

建议按下面的顺序讲：

1. “第一次运行先建立基线数据。”
2. “第二次运行沿用同样命令，系统会基于 CSV 快照做行级比对。”
3. “未变化记录会被识别为 `unchanged`，新增/变更记录才会写入。”
4. “运行台账 `crawl_runs.jsonl` 和变更台账 `crawl_changes.jsonl` 支持审计。”
5. “Harvard 还支持 `lastupdate` 水位线，能进一步减少无效抓取。”

## 8. 最小可用命令

如果你只想记一条最稳的命令，就用这个：

```bash
cd crawler
export SI_DATA_GOV_API_KEY="你的史密森尼API_KEY"
export HARVARD_ART_MUSEUMS_API_KEY="你的Harvard_API_KEY"
MUSEUMS=harvard CRAWL_LIMIT=20 ./run_incremental_update.sh
```

然后把它再执行一次，展示“第二次运行只处理变化项”。
